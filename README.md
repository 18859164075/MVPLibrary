##架构思路简介<br>
  mvp的基本思想这里不做过多解释，有兴趣的同学可以自己在各大博客学习一下。<br>
  比较深入学习过mvp架构的同学都知道，mvp现在比较大众的使用方法有两种：
  *1.使用activity作为view层，创建一个presenter和model与之对应，配合使用。
  *2.使用activity作为presenter层，创建一个view和model与之对应，配合使用。
